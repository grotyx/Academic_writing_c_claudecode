# Revision & Reviewer Response Guide (v0.3.0)

> 리뷰어 코멘트 대응 및 수정 원고 작성을 위한 가이드

---

## 개요

논문 투고 후 리뷰어 코멘트를 받으면:

1. **코멘트 전체를 먼저 읽는다** — 바로 답하지 않고 하루 이상 숙고
2. **공저자와 논의** — 대응 전략 수립
3. **Comment별 분류** — Major/Minor/Editorial 구분
4. **Point-by-point 응답서 작성**
5. **원고 수정** — 변경 사항 추적 (Track Changes)
6. **최종 검토** — 응답서 ↔ 수정 원고 일치 확인

---

## Response Letter 구조

### 기본 형식

```markdown
Dear Editor,

Thank you for the opportunity to revise our manuscript entitled "[TITLE]"
(Manuscript No. [NUMBER]). We appreciate the constructive comments from
the reviewers, which have helped us improve the manuscript. Below, we
provide a point-by-point response to each comment.

---

## Reviewer 1

### Comment 1.1
[리뷰어 코멘트 원문 그대로 복사]

**Response:**
[응답 내용]

**Changes made:**
[수정 내용 및 위치 — 페이지/줄번호 또는 섹션명]

### Comment 1.2
...

---

## Reviewer 2
...
```

### 형식 규칙

| 항목 | 규칙 |
|------|------|
| 리뷰어 코멘트 | 원문 그대로 복사 (수정 금지) |
| 응답 | 코멘트와 시각적으로 구분 (Bold, 색상, 들여쓰기 등) |
| 수정 위치 | 구체적으로 명시 ("Methods, paragraph 2", "Table 2", "Page 8, line 15") |
| 수정 텍스트 | 가능하면 변경된 문장을 응답서에 직접 인용 |
| 호칭 | 3인칭 사용 ("We agree with the reviewer..." ✅, "We agree with you..." ❌) |

## Comment 분류 및 대응 전략

### 분류 체계

| Category | 설명 | 대응 원칙 |
|----------|------|----------|
| **Major — Methodology** | 연구 설계, 통계, 대상군 관련 | 가장 먼저 대응, 추가 분석 필요 시 수행 |
| **Major — Interpretation** | 결과 해석, 결론 과장, 논리 관련 | 근거 기반으로 수정 또는 반론 |
| **Minor — Data/Presentation** | 테이블, 그림, 숫자 관련 | 즉시 수정 |
| **Minor — Writing** | 문법, 표현, 약어, 참고문헌 | 즉시 수정 |
| **Editorial** | 형식, 워드 수, 파일 형식 | 저널 규정대로 수정 |

### 대응 유형별 템플릿

#### 1. 동의하고 수정 (Agree & Revise) — 가장 흔함

```
We thank the reviewer for this insightful comment. We have revised
the [section] accordingly. Specifically, [description of change].
(See [location])
```

#### 2. 부분 동의 (Partial Agreement)

```
We appreciate the reviewer's suggestion. While we agree that [agreed part],
we believe [different part] because [evidence/reasoning]. To address the
reviewer's concern, we have [compromise action]. (See [location])
```

#### 3. 정중한 반론 (Respectful Disagreement)

```
We appreciate the reviewer raising this important point. After careful
consideration, we respectfully maintain our original approach because
[evidence-based reasoning]. [Optional: cite supporting literature].
However, we have added a discussion of this issue in the Limitations
section to acknowledge this perspective. (See [location])
```

#### 4. 할 수 없는 요청 (Cannot Implement)

```
We thank the reviewer for this suggestion. Unfortunately, [reason:
data not available / retrospective design limitation / beyond scope /
IRB constraints]. We acknowledge this as a limitation and have added
a statement in the Discussion section. (See [location])
```

#### 5. 오해 해소 (Clarify Misunderstanding)

```
We appreciate the reviewer's comment. We realize that our original
description may not have been sufficiently clear. To clarify, [explanation].
We have revised the relevant text to better convey this point.
(See [location])
```

## 외교적 표현 가이드

### 기본 원칙

1. **감사 → 입장 → 근거 → 조치** 순서로 작성
2. **리뷰어를 컨설턴트로 간주**, 적으로 보지 않음
3. **주요 거부는 최대 1-2개**로 제한 — 너무 많은 반론은 위험
4. **감정적 반응 금지** — 가혹한 리뷰를 받아도 하루 이상 숙고 후 작성

### 피해야 할 표현 vs 권장 표현

| 피해야 할 표현 ❌ | 권장 표현 ✅ |
|-----------------|------------|
| The reviewer is wrong/incorrect | We believe the reviewer may have a different perspective |
| We disagree | After careful consideration, we respectfully maintain... |
| This is already stated in the manuscript | We realize our original description may not have been sufficiently clear. We have revised... |
| We cannot do this | This is an excellent suggestion; however, due to [specific limitation]... |
| Obviously... / Clearly... | The data suggest that... |
| As we stated before... | To further clarify this point... |

### 추가 유용 표현

**동의 시:**
- "This is an excellent point that has helped strengthen our manuscript."
- "We agree with the reviewer and have made the following changes."

**추가 분석 수행 시:**
- "As suggested by the reviewer, we performed [additional analysis]. The results [support/do not change] our original findings."

**문헌 추가 시:**
- "We appreciate the reviewer's suggestion. We have added [N] additional references [refs] that address this point."

## Response Letter 체크리스트

### 작성 전
- [ ] 코멘트를 받은 후 최소 하루 이상 숙고했는가?
- [ ] 공저자와 대응 전략을 논의했는가?
- [ ] Comment를 분류했는가? (Major/Minor/Editorial)

### 작성 중
- [ ] 모든 코멘트에 빠짐없이 응답했는가?
- [ ] 리뷰어 코멘트 원문을 그대로 복사했는가?
- [ ] 응답이 시각적으로 구분되는가?
- [ ] 수정 위치가 구체적으로 명시되었는가?
- [ ] 3인칭으로 작성되었는가? ("the reviewer")
- [ ] 감정적/방어적 표현이 없는가?
- [ ] 거부한 항목에 대해 충분한 근거를 제시했는가?

### 작성 후
- [ ] 응답서의 모든 수정 사항이 원고에 반영되었는가?
- [ ] Track Changes가 활성화되어 있는가?
- [ ] 수정 원고의 줄번호/페이지가 응답서와 일치하는가?
- [ ] 공저자 전원이 최종 응답서를 검토했는가?

## QC Re-run for Revision

> Revision 제출 전에 QC를 재수행해야 함. 전체 Phase 6 QC (6 rounds)를 다시 할 필요는 없고, 변경 범위에 맞춰 선택적으로 실행.

### QC Re-run 체크리스트

| Round | 대상 범위 | 필수/선택 | 실행 조건 |
|-------|-----------|-----------|-----------|
| Round 1: Number consistency | 변경된 섹션 + Abstract/Tables (교차 검증) | **필수** | 항상 — 숫자 변경 시 연쇄 영향 확인 |
| Round 2: Reference verification | 새로 추가된 reference만 | **조건부** | 새 reference 추가 시 |
| Round 3: Logic & flow | 변경 섹션과 인접 섹션 (특히 Discussion ↔ Results) | **필수** | 항상 — 논리 흐름 파손 여부 확인 |
| Round 4: Terminology/abbreviation | 변경 섹션 + 새 용어 도입 여부 | **조건부** | 새 용어·약어 추가 시 |
| Round 5: Statistical quality | 재분석 결과만 | **조건부** | 리뷰어 요청으로 재분석 수행 시 |
| Round 6: Critical review | 전체 원고 | **선택** | Major revision에 한해 권장 |

### Revision QC 실행 원칙

1. **Round 1·3은 항상 재수행** — 최소 변경도 연쇄 영향 가능
2. **review/qc_log.md에 REV{N} 라운드 기록** — "QC Round 1 re-run for REV1, 2026-MM-DD, no discrepancies"
3. **변경된 숫자는 Abstract·Methods·Results·Discussion·Tables 5곳에서 교차 검증**
4. **응답서 제출 전 최종 Round 1 검증** — 응답서에 기록한 수정 숫자와 원고 숫자 일치

### Revision 후 체크리스트 (제출 전)

- [ ] Round 1 재수행 완료 (숫자 교차 검증)
- [ ] Round 3 재수행 완료 (논리 흐름)
- [ ] 새 reference 있으면 Round 2 재수행
- [ ] Response letter의 인용 line 번호가 수정본과 일치
- [ ] Tracked changes 파일 별도 준비 (저널 요구 시)
- [ ] QC log 업데이트 (review/qc_log.md)
